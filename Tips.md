# GitHub tips!!

**Try to learn GitHub if you haven't used it before.**

### *Some tips to keep in mind:*
- Always create a new branch for changes, so that the main branch remains stable.
- Write clear commit messages that explain the changes made.
- Regularly pull changes from the main branch to keep branch up to date.
- Use pull requests to propose changes and get feedback from other contributors.

### *To contribute to the NetMod-Repurposers project, follow these steps:*
1. Fork the repo to your GitHub Account.
2. Clone the forked repo to your local machine.
3. Create a new branch for your changes.
4. Make your changes and commit them with clear messages.
5. Push your changes to your forked repo on GitHub.
6. Open a pull request to the main repository, describing the changes you made and why they are beneficial.
7. Wait for feedback from the maintainers and be open to making revisions based on their suggestions
8. Once your pull request is approved, it will be merged into the main branch.
9. After your changes are merged, you can delete your branch to keep the repository clean.

### *Some useful Git commands:*
- `git clone <repo-url>`: Clone a repository to your local machine.
- `git checkout -b <branch-name>`: Create and switch to a new branch.
- `git add <file>`: Stage changes for commit.
- `git commit -m "commit message"`: Commit staged changes with a message.
- `git push origin <branch-name>`: Push your branch to the remote repository.
- `git pull origin main`: Pull the latest changes from the main branch to your current branch
- `git merge main`: Merge the main branch into your current branch to keep it up to date.
- `git branch -d <branch-name>`: Delete a local branch after it has been merged.
- `git push origin --delete <branch-name>`: Delete a remote branch after it has been merged.
